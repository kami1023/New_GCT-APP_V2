## 2025-05-15 - Search UX and Keyboard Accessibility
**Learning:** Replacing imperative DOM search logic with declarative React state ensures predictable rendering and enables clean integration of empty state UI (e.g., 'No results found'). Additionally, a '/' keyboard shortcut improves efficiency, but it must be hinted at in the placeholder for discoverability.
**Action:** Use React state for filtering collections, provide explicit 'No results found' UI with a clear action, and include keyboard shortcut hints in placeholders.
