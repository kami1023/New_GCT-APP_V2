## 2025-05-14 - Search UX & Accessibility Enhancement
**Learning:** Imperative DOM manipulation for filtering makes it difficult to provide accessible empty states and search feedback. Switching to declarative React state enables robust UI responses like "No results found" and "Clear Search" actions.
**Action:** Always prefer declarative state-based filtering for search interfaces to maintain a single source of truth and enable rich interaction patterns (empty states, loading indicators, keyboard shortcuts).
